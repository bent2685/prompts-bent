### 开发者记忆瓶

```
“我今天的工作到此为止。请为我写一封『给明早的自己的简信』：
1. 刚才工作的最后状态是什么？
2. 脑子里还没写出来的灵感或待办（Context）有哪些？
3. 明早坐下来后，输入的第一条指令建议是什么？
将这段内容保存到 `archives/CONTINUE_SOON.md`。
```

---

### “心流状态”断点保护

```
我现在需要离开一会儿。请总结当前 Session 的所有**未尽事宜**：
- **逻辑断点**：代码中哪一个函数还没写完？逻辑写到了哪一步？
- **上下文假设**：你现在大脑中缓存了哪些关于业务逻辑的临时假设？
- **启动指令**：当我回来时，我该输入哪条指令来立刻接续现在的心流？
请将此内容精简输出，方便我下次直接粘贴。
```

---

### 核心上下文摘要

```
我发现你开始忘记我们之前的某些约定。请立即扫描当前项目，并为我列出：
- **全局变量/状态**：目前哪些状态是单例或全局污染的？
- **隐含约束**：有哪些我们口头约定但没写在 Spec 里的规则？
- **技术偏好**：回顾之前的修改，总结我最喜欢的代码风格。
确认这些快照后，请将其作为你后续生成的‘底层准则’。
```
