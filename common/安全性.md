### 模拟黑客攻击

```
“假设你是一个试图攻击我这个系统的黑客，请审视我们刚才写的逻辑：
- 哪些 API 接口最容易被利用？
- 输入验证是否存在漏洞？是否有越权风险？
- 我们的错误处理是否泄露了系统内部路径或敏感配置？
- 请指出风险点并给出防御方案。
```

### 事务审计
```
请审查这段涉及多表操作的事务逻辑。
- **原子性检查**：在发生错误或 Panic 时，事务是否能确保 100% 回滚？
- **一致性风险**：在高并发下，是否存在『幻读』或『脏写』的可能？
- **事务粒度**：事务是否开启得太早或持有时间过长？建议如何缩小事务范围以提升并发性能。
- **错误映射**：针对数据库抛出的唯一约束冲突等错误，是否已经转化为了业务友好的错误码？
```
